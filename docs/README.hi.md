> 365 Open-Source Plan #020 · एक Windows ट्रे टूल: स्टार्टअप लॉन्चर · रिमाइंडर · सिस्टम स्टार्टअप आइटम · एक्शन समूह

![Clockwork](../assets/social-card.png)

एक छोटा Windows ट्रे टूल जो कंप्यूटर पर आपका दिन शुरू करने के दोहराव वाले हिस्सों को संभाल लेता है:

- 🚀 **स्टार्टअप सूची** — लॉगिन पर आपके रोज़मर्रा के ऐप्स क्रम से अपने आप खोलें (हर चरण के लिए एडमिन अधिकार, देरी, केवल कुछ वारों को / केवल N बजे से पहले, विंडो शैली, चल रहा हो तो सक्रिय करना, वैकल्पिक पथ), और साथ में कुछ छोटे-मोटे काम भी (विंडो बंद करना या सामने लाना, कीस्ट्रोक / टेक्स्ट भेजना, वॉल्यूम सेट करना…)।
- ⏰ **रिमाइंडर** — समय पर रिमाइंडर दिखाएँ; ज़ोर से पढ़ें; वार / हर N दिन / मासिक के अनुसार दोहराएँ; या "लॉगिन पर" ट्रिगर करें। **हाँ** क्लिक करने पर कोई प्रोग्राम चला सकते हैं, कोई फ़ाइल (जैसे संगीत) या URL खोल सकते हैं, या कोई एक्शन समूह चला सकते हैं।
- 🧹 **सिस्टम स्टार्टअप आइटम** — आपके PC पर **जो भी अपने आप शुरू होता है उन सबको** सूचीबद्ध करें और जो नहीं चाहिए उसे बंद करें (अक्षम, हटाया नहीं गया — कभी भी वापस चालू करें)। एक क्लिक किसी आइटम को आपकी अपनी स्टार्टअप सूची में "अधिग्रहण" कर लेता है।
- 🎛️ **एक्शन समूह** — क्रियाओं की एक श्रृंखला को पुनः उपयोग योग्य समूह में बाँधें (फोकस / मीटिंग / समाप्ति / सोने से पहले…) और ट्रे, एक **वैश्विक हॉटकी**, स्टार्टअप सूची, या किसी रिमाइंडर से एक क्लिक में चलाएँ। दिए गए टेम्पलेट शामिल हैं।

कोई इंस्टॉल नहीं, पूरी तरह पोर्टेबल एकल फ़ोल्डर, सब कुछ माउस से कॉन्फ़िगर करने योग्य; डार्क UI, हाई-DPI अनुकूल।

## आवश्यकताएँ

- Windows 10 / 11 (x64)
- कुछ भी इंस्टॉल करने की ज़रूरत नहीं: एक स्व-निहित एकल-फ़ाइल `Clockwork.exe` जिसमें .NET रनटाइम पहले से बंडल है।

## शुरुआत करना

1. [Releases](https://github.com/rockbenben/Clockwork/releases) से नवीनतम `Clockwork.exe` डाउनलोड करें और किसी भी फ़ोल्डर में रख दें (पोर्टेबल — जहाँ चाहें रखें)। खुद बनाना हो तो नीचे **डेवलपर्स के लिए** देखें।
2. सेटिंग विंडो खोलने के लिए **`Clockwork.exe`** पर डबल-क्लिक करें।
   - **पहली बार** चलाने पर यह एक **नमूना कॉन्फ़िगरेशन** लोड करता है (स्टार्टअप / रिमाइंडर / एक्शन समूह दिखाते हुए) ताकि आप उसे अपने हिसाब से बदल सकें। आपकी सेटिंग्स exe के बगल में `clockwork.settings.json` में रहती हैं — केवल स्थानीय, कभी कमिट नहीं होतीं।
3. हर बूट पर चलाने के लिए: **Settings** टैब पर, **लॉगिन पर शुरू करें** क्लिक करें (एडमिन अधिकारों के साथ एक शेड्यूल्ड टास्क पंजीकृत करता है, ताकि बूट पर UAC प्रॉम्प्ट की झड़ी न लगे)।

> यह चुपचाप ट्रे में बैठा रहता है। विंडो खोलने के लिए ट्रे आइकन पर डबल-क्लिक करें; विंडो का क्लोज़ बटन बस उसे ट्रे में छिपा देता है। पूरी तरह बंद करने के लिए ट्रे पर राइट-क्लिक कर **Exit** चुनें।

## पाँच टैब

### स्टार्टअप सूची

**चरणों की एक क्रमबद्ध सूची** जो लॉगिन पर ऊपर से नीचे चलती है। प्रकार चुनने के लिए **जोड़ें ▾** क्लिक करें; स्वतंत्र रूप से जोड़ें/हटाएँ/क्रम बदलें; हर चरण को सक्षम/अक्षम किया जा सकता है, **चरण-पश्चात देरी**, एक **दोहराव संख्या** (इसे N बार दोहराएँ), और शर्तें (**केवल कुछ वारों को / केवल N बजे से पहले**) दी जा सकती हैं। चरण के प्रकार:

- **प्रोग्राम लॉन्च करें** — लक्ष्य (फ़ाइल चुनने के लिए **ब्राउज़…**) / आर्ग्युमेंट / कार्य निर्देशिका (खाली छोड़ें = लक्ष्य का फ़ोल्डर) / एडमिन। लक्ष्य कोई `.exe`, दस्तावेज़, शॉर्टकट या URL हो सकता है; `.ps1` PowerShell के ज़रिए चलता है। उन्नत: **विंडो शैली** (न्यूनतम / अधिकतम / छिपी), **पहले से चल रहा हो तो सक्रिय करें** (फिर से लॉन्च करने के बजाय सामने लाएँ; प्रक्रिया नाम **चुनें…** से), **वैकल्पिक पथ** (प्रति पंक्ति एक पूर्ण पथ; पहला मौजूद पथ इस्तेमाल होता है — जब मशीनों में इंस्टॉल पथ अलग हों तब उपयोगी)।
- **कीज़ भेजें** — जैसे Win+D, Alt+K, Ctrl+Enter, F5 (शॉर्टकट को दबाकर रिकॉर्ड करने के लिए **कैप्चर**)।
- **टेक्स्ट भेजें** — फ़ोकस वाली विंडो में एक स्ट्रिंग टाइप करें (या **चुनें…** से चुनी गई **लक्ष्य प्रक्रिया** में)।
- **वॉल्यूम** — म्यूट / अनम्यूट / स्तर सेट करें।
- **विंडो क्रिया** — प्रक्रिया नाम से (**चुनें…**, खोज योग्य): बंद करें / न्यूनतम / अधिकतम / सामने लाएँ / सामने लाकर कीज़ भेजें; धीमे ऐप्स के लिए **विंडो दिखने का अधिकतम N सेकंड इंतज़ार** किया जा सकता है।
- **सिस्टम कमांड** — डेस्कटॉप दिखाएँ / लॉक करें / मॉनिटर बंद करें / रीसायकल बिन खाली करें / क्लिपबोर्ड साफ़ करें / Settings खोलें / Task Manager / स्क्रीनशॉट / स्लीप / हाइबरनेट / साइन आउट / रीस्टार्ट / शट डाउन (आख़िरी तीन पहले पुष्टि माँगते हैं)।
- **देरी** — अगले चरण से पहले बस N सेकंड रुकें।
- **एक्शन समूह** — कोई परिभाषित एक्शन समूह चलाएँ; पूरे समूह को दोहराने के लिए दोहराव संख्या सेट करें।

> **स्टार्टअप देरी** (Settings टैब, केवल बूट पर): लॉगिन के बाद एक निश्चित सेकंड इंतज़ार करें ताकि "लॉगिन तूफ़ान" (हर ऑटोस्टार्ट से डिस्क/CPU खींचतान) थमने के बाद सूची चले; मैन्युअल री-रन इससे प्रभावित नहीं होता। चीज़ें बहुत जल्दी शुरू हों तो इसे बढ़ाएँ (0–600 से)।

> **कभी भी रोकें** — ट्रे → **चल रही क्रियाएँ रोकें**, या वैश्विक **पैनिक हॉटकी** (Settings टैब पर सेट; डिफ़ॉल्ट `Ctrl+Alt+Q`)। जो भी चल रहा है वह मौजूदा क्रिया के बाद रुक जाता है; लंबी प्रतीक्षाएँ (स्टार्टअप देरी, विंडो का इंतज़ार) तुरंत बाधित हो जाती हैं।

### रिमाइंडर

एक **समय** सेट करें (या **लॉगिन पर** में बदलें), एक **आवृत्ति** (वार / हर N दिन / मासिक), और **टेक्स्ट**; चाहें तो ज़ोर से पढ़वाएँ। जिन रिमाइंडर में **हाँ-पर** क्रिया हो (प्रोग्राम चलाएँ / फ़ाइल खोलें / URL / एक्शन समूह चलाएँ) वे **हाँ / नहीं** डायलॉग दिखाते हैं जिसमें **स्नूज़** बटन होता है (डिफ़ॉल्ट 10 मिनट, ▾ मेनू 5–60 मिनट); बाकी कोने में एक **रिमाइंडर कार्ड** के रूप में आते हैं (तय सेकंड बाद अपने आप बंद, **0 = तब तक रहेगा जब तक आप बंद न करें**)। आप एक **मौन एक्शन समूह** भी सेट कर सकते हैं — बिना पॉपअप के समय पर कोई समूह चलाएँ।

उन्नत: **स्वतः बंद**, **बार-बार टोकना** (किसी समयसीमा तक हर N मिनट दोबारा दिखाना), **ट्रिगर-पश्चात देरी + यादृच्छिक अंतराल**, **छूट** (छोटे शटडाउन/स्लीप से छूटा ट्रिगर पकड़ लेना), **छूट जाए तो पूरा करें** (हाइबरनेशन/शटडाउन से छूटने पर एक बार दोबारा दिखाना), और हर N दिन के लिए एक **आधार तिथि** (**तिथि चुनें**)। "आज दिखाया गया" और "यहाँ तक स्नूज़ किया" स्थितियाँ रीस्टार्ट के बाद भी बनी रहती हैं (`clockwork.state.json`), इसलिए एक स्नूज़ रीस्टार्ट के पार भी टिकता है और कुछ भी दो बार नहीं दिखता।

फ़ोकस करना या मीटिंग लेनी है? ट्रे में **1 / 2 / 4 घंटे के लिए रिमाइंडर रोकें** (डू-नॉट-डिस्टर्ब) मिलता है: सब कुछ (मौन समूहों सहित) दबा दिया जाता है और समय पूरा होते ही अपने आप फिर शुरू हो जाता है।

### सिस्टम स्टार्टअप आइटम

**जो भी अपने आप शुरू होता है** उसे सूचीबद्ध करता है (registry Run कुंजियाँ, Startup फ़ोल्डर, शेड्यूल्ड टास्क)। किसी आइटम को बंद करने के लिए **सक्षम** अनचेक करें — **अक्षम, हटाया नहीं गया; बहाल करने के लिए फिर से चेक करें** (तुरंत प्रभावी)। **एडमिन ज़रूरी** के रूप में चिह्नित आइटम उन्नत अधिकारों के साथ फिर से खोलने का संकेत देते हैं। सिस्टम / नीति / एक-बार वाले आइटम (Group-Policy Run, RunOnce, Winlogon, Active Setup) सामान्य रूप से टॉगल नहीं हो सकते और **डिफ़ॉल्ट रूप से छिपे** होते हैं — उन्हें देखने के लिए **सिस्टम / केवल-पढ़ने योग्य आइटम दिखाएँ** टिक करें (धूसर)। **स्टार्टअप सूची में अधिग्रहण करें** किसी आइटम को Clockwork को सौंप देता है (केवल registry Run कुंजियाँ और Startup-फ़ोल्डर आइटम)। ऊपर एक **फ़िल्टर** नाम / कमांड से खोजता है; कटी हुई कमांड पर होवर करके उसे पूरा पढ़ें।

### एक्शन समूह

क्रियाओं को एक पुनः उपयोग योग्य समूह में बाँधें। **जोड़ें ▾** किसी **दिए गए टेम्पलेट** (फोकस / मीटिंग / समाप्ति / सोने से पहले / थोड़ी देर के लिए दूर / स्क्रीनशॉट) से एक समूह शुरू करता है — प्रक्रिया नाम बदलें और सहेजें। एक समूह **केवल क्रियाएँ परिभाषित करता है**; इसे चार तरीकों से ट्रिगर करें: ट्रे से (**चलाएँ: <समूह>**), एक **वैश्विक हॉटकी**, स्टार्टअप सूची में एक **एक्शन-समूह चरण** के रूप में (बूट पर), या किसी रिमाइंडर से (**हाँ-पर / मौन समूह**)। एक समूह एक समय में केवल एक ही प्रति चलाता है; एक **संदेश** चरण पुष्टि द्वार का काम कर सकता है (**नहीं** का उत्तर देने पर बाकी रद्द)।

> **वैश्विक हॉटकी** — समूह संपादक में हॉटकी बॉक्स पर क्लिक करें और एक शॉर्टकट (जैसे `Ctrl+Alt+F`) दबाएँ ताकि उस समूह को कहीं से भी चलाया जा सके, बिना किसी मेनू के। Esc रद्द करता है, Delete साफ़ करता है। अक्षम किए गए समूह अपना संयोजन छोड़ देते हैं; सिस्टम द्वारा आरक्षित संयोजन (Alt+F4, Ctrl+Shift+Esc…) और वे संयोजन जो पहले से किसी अन्य समूह या पैनिक हॉटकी द्वारा लिए गए हैं, एक सूचना के साथ अस्वीकार कर दिए जाते हैं।

### सेटिंग्स

**स्टार्टअप देरी** (0–600 से, केवल बूट पर), **शुरू होने पर ट्रे में न्यूनतम**, **पैनिक हॉटकी** (बॉक्स पर क्लिक कर अपना शॉर्टकट दबाएँ; Esc रद्द, Delete साफ़; डिफ़ॉल्ट `Ctrl+Alt+Q`), और **UI भाषा** (सरलीकृत चीनी, English, 日本語 और 15 और — कुल 18; भाषा बदलने पर लागू करने के लिए ऐप पुनः आरंभ होता है)।

## सुझाव

- **किसी पंक्ति को संपादित करने के लिए उस पर डबल-क्लिक करें।** पथ / प्रक्रिया / शॉर्टकट / तिथि भरते समय हाथ से टाइप करने की ज़रूरत नहीं: **ब्राउज़…**, **चुनें…** (खोज योग्य प्रक्रिया चयनक), **कैप्चर**, और **तिथि चुनें**।
- `Clockwork.exe` पर डबल-क्लिक केवल सेटिंग खोलता है — यह तुरंत स्टार्टअप सूची **नहीं** चलाता; उसके लिए ट्रे का **स्टार्टअप सूची फिर चलाएँ** इस्तेमाल करें।
- **इसे सामान्य तरीके से लॉन्च करें** (डबल-क्लिक / ट्रे / शेड्यूल्ड टास्क)। कुछ सैंडबॉक्स / सीमित-अधिकार वाले लॉन्चर निम्न-स्तरीय कॉल रोक देते हैं, इसलिए कीज़ भेजें / विंडो क्रियाएँ / चल रहा हो तो सक्रिय करें / प्रक्रिया में टेक्स्ट भेजें / वॉल्यूम काम नहीं कर सकते (आपको स्पष्ट सूचना मिलेगी; सादा "प्रोग्राम लॉन्च करें" प्रभावित नहीं होता)।
- आपकी कॉन्फ़िगरेशन `clockwork.settings.json` है (केवल स्थानीय)। नमूने पर रीसेट करने के लिए उसे हटा दें। रिमाइंडर स्थिति `clockwork.state.json` है (यह भी स्थानीय; हटाना सुरक्षित)।
- कोई `.ahk` चरण जोड़ने के लिए AutoHotkey इंस्टॉल होना चाहिए। वैश्विक हॉटकी / टेक्स्ट विस्तार दायरे से बाहर हैं — वह AutoHotkey की ताक़त है।

## डेवलपर्स के लिए

C#/.NET WPF; स्रोत `app/` में (`.NET 10 SDK` चाहिए)। परतें: `Core/` शुद्ध लॉजिक · `Native/` Win32 इंटरॉप · `Engine/` निष्पादन · `ViewModels/` + `Views/` UI · `I18n/` + `Resources/` स्थानीयकरण (न्यूट्रल = चीनी स्रोत, हर भाषा के लिए एक `Strings.<code>.resx` सैटेलाइट)।

- टेस्ट चलाएँ (xUnit):
  ```powershell
  dotnet test app.Tests/Clockwork.Tests.csproj
  ```
- स्व-निहित एकल-फ़ाइल exe बनाएँ (single-file / self-contained / कम्प्रेशन गुण csproj में सेट हैं):
  ```powershell
  dotnet publish app/Clockwork.csproj -c Release -r win-x64
  ```
  आउटपुट: `app/bin/Release/net10.0-windows/win-x64/publish/Clockwork.exe`।
- **CI / रिलीज़** (GitHub Actions): push / PR एक Windows runner पर बिल्ड करता है और सभी टेस्ट चलाता है; एक `v*` टैग पुश करना (जैसे `v2.0.0`) बिल्ड करता है, टैग से फ़ाइल संस्करण अंकित करता है, एक GitHub Release बनाता है और `Clockwork.exe` संलग्न करता है।

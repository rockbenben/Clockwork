> 365 開源計畫 #020 · 一款 Windows 工作列小工具：啟動啟動器 · 提醒 · 系統開機自啟項 · 動作群組

![Clockwork](../assets/social-card.png)

一款小巧的 Windows 工作列工具，替你打理每天坐到電腦前那些例行公事：

- 🚀 **啟動清單** — 登入時依序自動開啟你每天要用的應用程式（可為每個步驟設定系統管理員權限、延遲、僅限特定星期 / 僅限某點之前、視窗樣式、若已在執行則啟動、備援路徑），順手還能做幾件雜事（關閉或聚焦視窗、送出按鍵 / 文字、設定音量……）。
- ⏰ **提醒** — 準時彈出提醒；朗讀出來；按星期 / 每 N 天 / 每月重複；或「登入時」觸發。點擊 **是** 可以執行程式、開啟檔案（例如音樂）或網址，或執行一個動作群組。
- 🧹 **系統開機自啟項** — 列出**你電腦上所有會自動啟動的項目**，把不需要的關掉（是停用，而非刪除——隨時可再開回來）。一鍵即可「接管」某個項目，把它併入你自己的啟動清單。
- 🎛️ **動作群組** — 把一連串動作打包成一個可重複使用的群組（專注 / 會議 / 收尾 / 就寢……），從工作列、**全域熱鍵**、啟動清單或提醒中一鍵觸發。內建範本可用。

免安裝、完全可攜的單一資料夾、一切都能用滑鼠設定；深色介面、支援高 DPI。

> 📖 **詳細使用說明：** [English](USAGE.md) · [中文](USAGE.zh-CN.md)

## 系統需求

- Windows 10 / 11（x64）
- 無需安裝：一個自帶 .NET 執行環境、自包含的單一檔案 `Clockwork.exe`。

## 開始使用

1. 從 [Releases](https://github.com/rockbenben/Clockwork/releases) 下載最新的 `Clockwork-<版本號>.zip`，解壓後裡面是單一個 `Clockwork.exe`；把它丟進任意資料夾（可攜——放哪都行）。想自行建置，請見下方 **給開發者**。
2. 雙擊 **`Clockwork.exe`** 開啟設定視窗。
   - **首次執行**時會載入一份**範例設定**（示範啟動 / 提醒 / 動作群組），方便你改成自己的。你的設定存放在 exe 旁邊的 `clockwork.settings.json`——僅存本機，絕不上傳。
3. 若要每次開機都執行：在 **設定** 分頁點 **登入時啟動**（會以系統管理員權限註冊一個排程工作，因此開機時不會冒出一堆 UAC 提示）。

> 它會安靜地待在工作列。雙擊工作列圖示即可開啟視窗；視窗的關閉按鈕只是把它縮回工作列。要真正結束，請用工作列圖示的右鍵 **結束**。

## 五個分頁

### 啟動清單

一份**有順序的步驟清單**，登入時由上到下執行。點 **新增 ▾** 選擇類型；可自由新增 / 移除 / 重新排序；每個步驟都能啟用 / 停用、設定**步驟後延遲**、**重複次數**（迴圈執行 N 次），以及條件（**僅限特定星期 / 僅限某點之前**）。步驟類型：

- **啟動程式** — 目標（**瀏覽…** 選擇檔案）/ 參數 / 工作目錄（留空 = 目標所在資料夾）/ 系統管理員。目標可以是 `.exe`、文件、捷徑或網址；`.ps1` 會透過 PowerShell 執行。進階：**視窗樣式**（最小化 / 最大化 / 隱藏）、**若已在執行則啟動**（把它帶到前景而非重新啟動；行程名稱可用 **挑選…**）、**備援路徑**（一行一個完整路徑；使用第一個存在的——不同機器安裝路徑不同時很方便）。
- **送出按鍵** — 例如 Win+D、Alt+K、Ctrl+Enter、F5（**擷取** 可按下快速鍵直接錄下來）。
- **送出文字** — 把一段字串輸入到聚焦的視窗（或透過 **挑選…** 選定的**目標行程**）。
- **音量** — 靜音 / 取消靜音 / 設定音量。
- **視窗動作** — 依行程名稱（**挑選…**，可搜尋）：關閉 / 最小化 / 最大化 / 帶到前景 / 帶到前景並送出按鍵；較慢的應用程式可**最多等待 N 秒讓視窗出現**。
- **系統命令** — 顯示桌面 / 鎖定 / 關閉螢幕 / 清空資源回收筒 / 清除剪貼簿 / 開啟設定 / 工作管理員 / 螢幕截圖 / 睡眠 / 休眠 / 登出 / 重新開機 / 關機（最後三項會先確認）。
- **延遲** — 在下一步之前單純等待 N 秒。
- **動作群組** — 執行一個已定義的動作群組；設定重複次數可讓整個群組迴圈執行。

> **啟動延遲**（設定分頁，僅開機時）：登入後固定等待若干秒，讓「登入風暴」（所有自啟項造成的磁碟 / CPU 爭用）過去後再執行清單；手動重新執行不受影響。若東西啟動得太早，就把它調高（0–600 秒）。

> **隨時停止** — 工作列 → **停止執行中的動作**，或全域**緊急停止熱鍵**（在設定分頁設定；預設 `Ctrl+Alt+Q`）。正在執行的東西會在目前動作結束後停止；長時間的等待（啟動延遲、等待視窗）會立即中斷。

### 提醒

設定一個**時間**（或切換為**登入時**）、一種**重複方式**（星期 / 每 N 天 / 每月），以及**文字**；可選擇朗讀出來。帶有 **點「是」動作**（執行程式 / 開啟檔案 / 網址 / 執行動作群組）的提醒會彈出一個 **是 / 否** 對話框並附 **延後** 按鈕（預設 10 分鐘，▾ 選單可選 5–60 分鐘）；其餘的則以**提醒卡片**從角落滑入（在設定的秒數後自動關閉，**0 = 保持顯示直到你手動關閉**）。你也可以設定一個**靜默動作群組**——準時執行一個群組而不彈出任何視窗。

進階：**自動關閉**、**重複催促**（每 N 分鐘再彈一次，直到期限）、**觸發後延遲 + 隨機抖動**、**寬限**（補上因短暫關機 / 睡眠而錯過的一次觸發）、**錯過則補觸發**（休眠 / 關機略過後補觸發一次），以及每 N 天的**錨定日期**（**挑選日期**）。「今日已觸發」與「延後至」會在重新啟動後保留（`clockwork.state.json`），因此延後會延續到重新啟動之後，且不會重複觸發。

需要專注或要開會？工作列提供 **暫停提醒 1 / 2 / 4 小時**（勿擾模式）：一切（包括靜默群組）都會被抑制，時間到後自動恢復。

### 系統開機自啟項

列出**所有會自動啟動的項目**（登錄檔 Run 機碼、啟動資料夾、排程工作）。取消勾選 **啟用** 即可關掉某個項目——**是停用，而非刪除；重新勾選即可還原**（立即生效）。標示 **需要系統管理員** 的項目會提示以提升權限重新啟動。系統 / 原則 / 一次性項目（群組原則 Run、RunOnce、Winlogon、Active Setup）無法動它，且**預設隱藏**——勾選 **顯示系統 / 唯讀項目** 即可查看（呈灰色）。**在某一列上按右鍵**可選 **接管到啟動清單**（把該項目交給 Clockwork；僅限登錄檔 Run 機碼與啟動資料夾項目）或 **從系統中刪除**（**徹底移除該項目——會先確認，且無法復原**；只是想讓它別開機啟動的話，取消勾選才是可還原的做法）。頂部的**篩選**可依名稱 / 命令搜尋；把滑鼠移到被截斷的命令上即可看到完整內容。

### 動作群組

把一連串動作打包成一個可重複使用的群組。**新增 ▾** 會從**內建範本**（專注 / 會議 / 收尾 / 就寢 / 暫時離開 / 螢幕截圖）起手——調整行程名稱後儲存即可。一個群組**只定義動作**；有四種方式觸發它：從工作列（**執行：<群組>**）、**全域熱鍵**、作為啟動清單裡的**動作群組步驟**（開機時），或從提醒（**點「是」/ 靜默群組**）。一個群組同時只會執行一份；**訊息** 步驟可作為確認關卡（回答 **否** 會中止其餘部分）。

> **全域熱鍵** — 在群組編輯器中點熱鍵方框並按下快速鍵（例如 `Ctrl+Alt+F`），即可從任何地方執行該群組，無需開啟選單。Esc 取消、Delete 清除。停用的群組會釋放其組合鍵；系統保留的組合鍵（Alt+F4、Ctrl+Shift+Esc…）以及已被其他群組或緊急停止熱鍵佔用的組合鍵會被拒絕並提示。

### 設定

**啟動延遲**（0–600 秒，僅開機時）、**啟動時最小化到工作列**、**緊急停止熱鍵**（點方框並按下你的快速鍵；Esc 取消，Delete 清除；預設 `Ctrl+Alt+Q`），以及 **介面語言**（簡體中文、English、日本語 等共 18 種；切換會重新啟動應用程式以套用）。

**匯出設定 / 匯入設定** — 把整套設定搬到另一台電腦，或留一份備份。匯出會把 `clockwork.settings.json` 複製一份到你指定的任何位置；匯入則會取代**全部內容**（啟動清單 / 提醒 / 動作群組 / 設定），因此會先確認、把目前的設定備份為 `clockwork.settings.json.bak`，再重新啟動應用程式以套用。

## 小技巧

- **雙擊某一列即可編輯**。填寫路徑 / 行程 / 快速鍵 / 日期時不必手動輸入：有 **瀏覽…**、**挑選…**（可搜尋的行程挑選器）、**擷取** 與 **挑選日期**。
- **複製**（提醒 / 動作群組分頁）會把選取的那一列複製一份到它下方——比從頭重建一個幾乎一樣的快多了；複製出來的群組會命名為「…（副本）」。
- **刪除一律會先詢問**，處處如此——清單列、群組編輯器裡的步驟，以及系統開機自啟項。
- 雙擊 `Clockwork.exe` 只會開啟設定——它**不會**立刻執行啟動清單；要執行請用工作列的 **重新執行啟動清單**。
- **以正常方式啟動它**（雙擊 / 工作列 / 排程工作）。某些沙箱 / 降權啟動器會封鎖低階呼叫，因此送出按鍵 / 視窗動作 / 若已在執行則啟動 / 送文字到行程 / 音量可能無法運作（你會收到明確的提示；單純的「啟動程式」不受影響）。
- 你的設定是 `clockwork.settings.json`（僅存本機）。刪掉它即可重設回範例。提醒狀態是 `clockwork.state.json`（也是本機；可安全刪除）。
- 新增 `.ahk` 步驟需要已安裝 AutoHotkey。全域熱鍵 / 文字展開不在本工具範圍內——那是 AutoHotkey 的強項。

## 給開發者

C#/.NET WPF；原始碼在 `app/`（需要 .NET 10 SDK）。分層：`Core/` 純邏輯 · `Native/` Win32 互通 · `Engine/` 執行 · `ViewModels/` + `Views/` UI · `I18n/` + `Resources/` 在地化（中性 = 中文原文，每種語言一個 `Strings.<code>.resx` 附屬檔）。

- 執行測試（xUnit）：
  ```powershell
  dotnet test app.Tests/Clockwork.Tests.csproj
  ```
- 建置自包含的單一檔案 exe（單一檔案 / 自包含 / 壓縮已在 csproj 中設定）：
  ```powershell
  dotnet publish app/Clockwork.csproj -c Release -r win-x64
  ```
  輸出：`app/bin/Release/net10.0-windows/win-x64/publish/Clockwork.exe`。
- **CI / 發行**（GitHub Actions）：push / PR 會在 Windows runner 上建置並執行所有測試；推送 `v*` 標籤（例如 `v2.0.0`）會建置、依標籤標記檔案版本、建立 GitHub Release 並附上 `Clockwork-<標籤>.zip`（內含 `Clockwork.exe`）。

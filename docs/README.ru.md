> Открытый план 365 #020 · Утилита в трее Windows: запуск при старте · напоминания · элементы автозагрузки системы · группы действий

![Clockwork](../assets/social-card.png)

Небольшая утилита в трее Windows, которая берёт на себя рутину при начале рабочего дня за компьютером:

- 🚀 **Список автозапуска** — автоматически открывает ваши повседневные приложения при входе, по порядку (права администратора для каждого шага, задержки, только-в-определённые-дни-недели / только-до-N-часов, стиль окна, активировать-если-запущено, запасные пути) и попутно выполняет мелкие задачи (закрыть или сфокусировать окна, отправить нажатия клавиш / текст, задать громкость…).
- ⏰ **Напоминания** — показывают напоминание вовремя; проговаривают его вслух; повторяют по дням недели / каждые-N-дней / ежемесячно; либо срабатывают «при входе». Нажатие **Да** может запустить программу, открыть файл (например, музыку) или URL либо выполнить группу действий.
- 🧹 **Элементы автозагрузки системы** — показывают **всё, что автоматически запускается на вашем ПК**, и отключают то, что вам не нужно (отключено, а не удалено — можно вернуть в любой момент). Один клик «перехватывает» элемент в ваш собственный список автозапуска.
- 🎛️ **Группы действий** — объедините ряд действий в многоразовую группу (Фокус / Совещание / Завершение / Отход ко сну…) и запускайте её одним кликом из трея, списка автозапуска или напоминания. Встроенные шаблоны включены.

Без установки, полностью портативна в одной папке, всё настраивается мышью; тёмный интерфейс, поддержка высокого DPI.

## Требования

- Windows 10 / 11 (x64)
- Ничего устанавливать не нужно: автономный однофайловый `Clockwork.exe` со встроенной средой выполнения .NET.

## Начало работы

1. Скачайте свежий `Clockwork.exe` из раздела [Releases](https://github.com/rockbenben/Clockwork/releases) и поместите его в любую папку (портативно — куда угодно). Чтобы собрать самому, см. **Для разработчиков** ниже.
2. Дважды щёлкните по **`Clockwork.exe`**, чтобы открыть окно настроек.
   - При **первом запуске** загружается **пример конфигурации** (демонстрирующий автозапуск / напоминания / группы действий), чтобы вы адаптировали его под себя. Ваши настройки хранятся в `clockwork.settings.json` рядом с exe — только локально, никогда не коммитятся.
3. Чтобы запускать его при каждой загрузке: на вкладке **Настройки** нажмите **Запускать при входе** (регистрирует запланированную задачу с правами администратора, поэтому при загрузке не будет вала запросов UAC).

> Она тихо сидит в трее. Дважды щёлкните по значку в трее, чтобы открыть окно; кнопка закрытия окна лишь сворачивает его в трей. Чтобы выйти по-настоящему, используйте **Выход** в контекстном меню трея.

## Пять вкладок

### Список автозапуска
**Упорядоченный список шагов**, выполняемых сверху вниз при входе. Нажмите **Добавить ▾**, чтобы выбрать тип; добавляйте/удаляйте/меняйте порядок свободно; каждый шаг можно включить/отключить, задать ему **задержку после шага**, **число повторов** (повторить N раз) и условия (**только в определённые дни недели / только до N часов**). Типы шагов:

- **Запуск программы** — цель (**Обзор…** для выбора файла) / аргументы / рабочий каталог (оставьте пустым = папка цели) / администратор. Целью может быть `.exe`, документ, ярлык или URL; `.ps1` выполняется через PowerShell. Дополнительно: **стиль окна** (свёрнутое / развёрнутое / скрытое), **активировать, если уже запущено** (вывести на передний план вместо перезапуска; имя процесса через **Выбрать…**), **запасные пути** (по одному полному пути в строке; используется первый существующий — удобно, когда пути установки различаются на разных машинах).
- **Отправка клавиш** — например, Win+D, Alt+K, Ctrl+Enter, F5 (**Захват** для записи сочетания нажатием).
- **Отправка текста** — вводит строку в активное окно (или в выбранный **целевой процесс** через **Выбрать…**).
- **Громкость** — отключить / включить звук / задать уровень.
- **Действие с окном** — по имени процесса (**Выбрать…**, с поиском): закрыть / свернуть / развернуть / вывести-на-передний-план / вывести-на-передний-план-и-отправить-клавиши; медленные приложения могут **ждать появления окна до N секунд**.
- **Системная команда** — показать рабочий стол / заблокировать / выключить монитор / очистить корзину / очистить буфер обмена / открыть Параметры / Диспетчер задач / снимок экрана / сон / гибернация / выход из системы / перезагрузка / выключение (последние три сначала запрашивают подтверждение).
- **Задержка** — просто подождать N секунд перед следующим шагом.
- **Группа действий** — выполнить заданную группу действий; задайте число повторов, чтобы повторить всю группу.

> **Задержка при старте** (вкладка Настройки, только при загрузке): подождать фиксированное число секунд после входа, чтобы «шторм входа» (конкуренция за диск/ЦП со стороны всех автозапусков) прошёл до выполнения списка; ручной повторный запуск не затрагивается. Увеличьте её (0–600 с), если всё стартует слишком рано.

> **Остановить в любой момент** — трей → **Остановить выполняемые действия**, либо глобальная **аварийная горячая клавиша** (задаётся на вкладке Настройки; по умолчанию `Ctrl+Alt+Q`). То, что выполняется, останавливается после текущего действия; долгие ожидания (задержка при старте, ожидание окна) прерываются немедленно.

### Напоминания
Задайте **время** (или переключитесь на **при входе**), **периодичность** (дни недели / каждые-N-дней / ежемесячно) и **текст**; при желании — проговаривание вслух. Напоминания с действием **По-Да** (запустить программу / открыть файл / URL / выполнить группу действий) показывают диалог **Да / Нет** с кнопкой **Отложить** (по умолчанию 10 мин, меню ▾ 5–60 мин); остальные выезжают **карточкой напоминания** в углу (автозакрытие через заданное число секунд, **0 = остаётся, пока вы её не закроете**). Можно также задать **тихую группу действий** — выполнять группу вовремя без всплывающего окна.

Дополнительно: **автозакрытие**, **повторное напоминание** (всплывает снова каждые N минут до дедлайна), **задержка после срабатывания + случайный разброс**, **допуск** (подхватить срабатывание, пропущенное из-за кратковременного выключения/сна), **наверстать при пропуске** (сработать ещё раз, если гибернация/выключение его пропустили) и **опорная дата** для «каждые N дней» (**Выбрать дату**). «Сработало сегодня» и «отложено до» переживают перезапуски (`clockwork.state.json`), поэтому отложенное переносится через перезапуск и ничто не срабатывает дважды.

Нужно сосредоточиться или провести совещание? Трей предлагает **Приостановить напоминания на 1 / 2 / 4 часа** (Не беспокоить): всё (включая тихие группы) подавляется и автоматически возобновляется по истечении времени.

### Элементы автозагрузки системы
Перечисляет **всё, что автоматически запускается** (ключи Run в реестре, папки Автозагрузки, запланированные задачи). Снимите флажок **Включить**, чтобы отключить элемент — **отключено, а не удалено; поставьте флажок снова, чтобы восстановить** (действует немедленно). Элементы, помеченные **нужны права администратора**, предлагают перезапуститься с повышением. Системные / политики / одноразовые элементы (Run групповой политики, RunOnce, Winlogon, Active Setup) нельзя переключать обычным образом, и они **скрыты по умолчанию** — отметьте **Показывать системные / только-для-чтения элементы**, чтобы их увидеть (недоступны, приглушены). **Перехватить в список автозапуска** передаёт элемент Clockwork (только ключи Run в реестре и элементы папки Автозагрузки). Верхний **фильтр** ищет по имени / команде; наведите курсор на усечённую команду, чтобы прочитать её целиком.

### Группы действий
Объедините действия в многоразовую группу. **Добавить ▾** создаёт её из **встроенного шаблона** (Фокус / Совещание / Завершение / Отход ко сну / Отлучиться / Снимок экрана) — подправьте имена процессов и сохраните. Группа **только определяет действия**; запускайте её тремя способами: из трея (**Запустить: <группа>**), как **шаг группы действий** в списке автозапуска (при загрузке) или из напоминания (**По-Да / тихая группа**). Одновременно выполняется только одна копия группы; шаг **сообщения** может служить барьером подтверждения (ответ **Нет** прерывает остальное).

### Настройки
**Задержка при старте** (0–600 с, только при загрузке), **запускать свёрнутой в трей**, **аварийная горячая клавиша** (щёлкните по полю и нажмите ваше сочетание; Esc отменяет, Delete очищает; по умолчанию `Ctrl+Alt+Q`) и **язык интерфейса** (упрощённый китайский, английский, 日本語 и ещё 15 — всего 18; переключение перезапускает приложение для применения).

## Советы

- **Дважды щёлкните по строке, чтобы её отредактировать.** При заполнении путей / процессов / сочетаний / дат не нужно вводить вручную: **Обзор…**, **Выбрать…** (выбор процесса с поиском), **Захват** и **Выбрать дату**.
- Двойной щелчок по `Clockwork.exe` только открывает настройки — он **не** запускает сразу список автозапуска; для этого используйте **Повторно выполнить список автозапуска** в трее.
- **Запускайте её обычным способом** (двойной щелчок / трей / запланированная задача). Некоторые песочницы / запускающие с пониженными правами блокируют низкоуровневые вызовы, поэтому отправка клавиш / действия с окном / активировать-если-запущено / отправка-текста-в-процесс / громкость могут не работать (вы получите чёткое уведомление; обычный «запуск программы» не затрагивается).
- Ваша конфигурация — `clockwork.settings.json` (только локально). Удалите её, чтобы сбросить к примеру. Состояние напоминаний — `clockwork.state.json` (тоже локально; удалять безопасно).
- Для добавления шага `.ahk` нужен установленный AutoHotkey. Глобальные горячие клавиши / расширение текста вне области задач — это сильная сторона AutoHotkey.

## Для разработчиков

C#/.NET WPF; исходники в `app/` (нужен .NET 10 SDK). Слои: `Core/` чистая логика · `Native/` interop Win32 · `Engine/` выполнение · `ViewModels/` + `Views/` интерфейс · `I18n/` + `Resources/` локализация (нейтральный = китайский источник, по одному сателлиту `Strings.<code>.resx` на язык).

- Запуск тестов (xUnit):
  ```powershell
  dotnet test app.Tests/Clockwork.Tests.csproj
  ```
- Сборка автономного однофайлового exe (однофайловость / автономность / сжатие заданы в csproj):
  ```powershell
  dotnet publish app/Clockwork.csproj -c Release -r win-x64
  ```
  Результат: `app/bin/Release/net10.0-windows/win-x64/publish/Clockwork.exe`.
- **CI / релизы** (GitHub Actions): push / PR собирают и прогоняют все тесты на Windows-раннере; отправка тега `v*` (например, `v2.0.0`) собирает, проставляет версию файла из тега, создаёт GitHub Release и прикрепляет `Clockwork.exe`.

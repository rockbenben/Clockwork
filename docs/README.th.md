> 365 Open-Source Plan #020 · เครื่องมือใน System Tray ของ Windows: ตัวเปิดโปรแกรมตอนเริ่มระบบ · การเตือน · รายการเริ่มต้นของระบบ · กลุ่มการทำงาน

![Clockwork](../assets/social-card.png)

เครื่องมือเล็ก ๆ ใน System Tray ของ Windows ที่ช่วยจัดการส่วนงานซ้ำ ๆ ในตอนเริ่มต้นวันทำงานหน้าคอมพิวเตอร์ของคุณ:

- 🚀 **รายการเริ่มต้น** — เปิดแอปที่คุณใช้ประจำโดยอัตโนมัติเมื่อล็อกอิน ตามลำดับ (สิทธิ์ผู้ดูแลระบบรายขั้นตอน, การหน่วงเวลา, เฉพาะบางวันในสัปดาห์ / เฉพาะก่อน N นาฬิกา, สไตล์หน้าต่าง, สลับไปหน้าต่างเดิมถ้ากำลังรันอยู่, เส้นทางสำรอง) และทำงานเล็ก ๆ น้อย ๆ ระหว่างทาง (ปิดหรือโฟกัสหน้าต่าง, ส่งคีย์ / ข้อความ, ตั้งระดับเสียง…)
- ⏰ **การเตือน** — เด้งการเตือนตรงเวลา; อ่านออกเสียง; ทำซ้ำตามวันในสัปดาห์ / ทุก ๆ N วัน / รายเดือน; หรือทริกเกอร์ "เมื่อล็อกอิน" การคลิก **ใช่** สามารถรันโปรแกรม, เปิดไฟล์ (เช่น เพลง) หรือ URL, หรือรันกลุ่มการทำงาน
- 🧹 **รายการเริ่มต้นของระบบ** — แสดง **ทุกอย่างบนพีซีของคุณที่เริ่มทำงานอัตโนมัติ** และปิดสิ่งที่คุณไม่ต้องการ (ปิดใช้งาน ไม่ได้ลบ — เปิดกลับได้ทุกเมื่อ) คลิกเดียวเพื่อ "รับช่วง" รายการเข้าสู่รายการเริ่มต้นของคุณเอง
- 🎛️ **กลุ่มการทำงาน** — รวมชุดการทำงานหลายอย่างไว้ในกลุ่มที่นำกลับมาใช้ซ้ำได้ (โฟกัส / ประชุม / สรุปงาน / เข้านอน…) และทริกเกอร์ด้วยการคลิกครั้งเดียวจาก System Tray, **คีย์ลัดส่วนกลาง**, รายการเริ่มต้น, หรือการเตือน มีเทมเพลตในตัวให้พร้อมใช้

ไม่ต้องติดตั้ง พกพาได้เต็มรูปแบบในโฟลเดอร์เดียว ตั้งค่าทุกอย่างได้ด้วยเมาส์ ธีมมืด รองรับหน้าจอความละเอียดสูง (high-DPI)

## ความต้องการของระบบ

- Windows 10 / 11 (x64)
- ไม่ต้องติดตั้งอะไร: ไฟล์เดียวแบบ self-contained `Clockwork.exe` ที่มี .NET runtime รวมมาให้ในตัว

## เริ่มต้นใช้งาน

1. ดาวน์โหลด `Clockwork.exe` เวอร์ชันล่าสุดจาก [Releases](https://github.com/rockbenben/Clockwork/releases) แล้ววางไว้ในโฟลเดอร์ใดก็ได้ (พกพาได้ — วางที่ไหนก็ได้) หากต้องการบิลด์เอง ดู **สำหรับนักพัฒนา** ด้านล่าง
2. ดับเบิลคลิก **`Clockwork.exe`** เพื่อเปิดหน้าต่างการตั้งค่า
   - ในการ **รันครั้งแรก** จะโหลด **คอนฟิกตัวอย่าง** (สาธิตรายการเริ่มต้น / การเตือน / กลุ่มการทำงาน) เพื่อให้คุณปรับให้เข้ากับของตัวเอง การตั้งค่าของคุณจะอยู่ใน `clockwork.settings.json` ข้าง ๆ ไฟล์ exe — อยู่ในเครื่องเท่านั้น ไม่เคยถูก commit
3. เพื่อให้รันทุกครั้งที่บูต: ที่แท็บ **การตั้งค่า** คลิก **เริ่มเมื่อล็อกอิน** (ลงทะเบียนงานที่ตั้งเวลาไว้ด้วยสิทธิ์ผู้ดูแลระบบ จึงไม่มีหน้าต่าง UAC เด้งเป็นพรืดตอนบูต)

> มันจะอยู่เงียบ ๆ ใน System Tray ดับเบิลคลิกไอคอนใน Tray เพื่อเปิดหน้าต่าง; ปุ่มปิดของหน้าต่างเพียงซ่อนมันลง Tray เท่านั้น หากต้องการปิดจริง ๆ ให้คลิกขวาที่ Tray แล้วเลือก **ออก**

## ห้าแท็บ

### รายการเริ่มต้น

**รายการขั้นตอนที่มีลำดับ** ซึ่งจะรันจากบนลงล่างเมื่อล็อกอิน คลิก **เพิ่ม ▾** เพื่อเลือกประเภท; เพิ่ม/ลบ/จัดลำดับใหม่ได้อย่างอิสระ; แต่ละขั้นตอนสามารถเปิด/ปิดใช้งาน, กำหนด **การหน่วงเวลาหลังขั้นตอน**, **จำนวนครั้งที่ทำซ้ำ** (วนซ้ำ N ครั้ง) และเงื่อนไข (**เฉพาะบางวันในสัปดาห์ / เฉพาะก่อน N นาฬิกา**) ประเภทของขั้นตอน:

- **เปิดโปรแกรม** — เป้าหมาย (**เรียกดู…** เพื่อเลือกไฟล์) / อาร์กิวเมนต์ / โฟลเดอร์ทำงาน (เว้นว่าง = โฟลเดอร์ของเป้าหมาย) / สิทธิ์ผู้ดูแลระบบ เป้าหมายอาจเป็น `.exe`, เอกสาร, ทางลัด หรือ URL; ไฟล์ `.ps1` จะรันผ่าน PowerShell ขั้นสูง: **สไตล์หน้าต่าง** (ย่อเล็กสุด / ขยายใหญ่สุด / ซ่อน), **สลับไปหน้าต่างเดิมถ้ากำลังรันอยู่** (นำมาไว้ด้านหน้าแทนที่จะเปิดใหม่; ระบุชื่อโปรเซสผ่าน **เลือก…**), **เส้นทางสำรอง** (หนึ่งเส้นทางเต็มต่อบรรทัด; ใช้เส้นทางแรกที่มีอยู่จริง — สะดวกเมื่อเส้นทางติดตั้งต่างกันในแต่ละเครื่อง)
- **ส่งคีย์** — เช่น Win+D, Alt+K, Ctrl+Enter, F5 (**บันทึก** เพื่อบันทึกคีย์ลัดด้วยการกดจริง)
- **ส่งข้อความ** — พิมพ์สตริงลงในหน้าต่างที่โฟกัสอยู่ (หรือ **โปรเซสเป้าหมาย** ที่เลือกไว้ผ่าน **เลือก…**)
- **ระดับเสียง** — ปิดเสียง / เปิดเสียง / ตั้งระดับ
- **การกระทำกับหน้าต่าง** — ตามชื่อโปรเซส (**เลือก…**, ค้นหาได้): ปิด / ย่อเล็กสุด / ขยายใหญ่สุด / นำมาไว้ด้านหน้า / นำมาไว้ด้านหน้าแล้วส่งคีย์; แอปที่ทำงานช้าสามารถ **รอนานถึง N วินาทีให้หน้าต่างปรากฏ**
- **คำสั่งระบบ** — แสดงเดสก์ท็อป / ล็อก / ปิดจอภาพ / ล้างถังรีไซเคิล / ล้างคลิปบอร์ด / เปิดการตั้งค่า / Task Manager / ภาพหน้าจอ / สลีป / ไฮเบอร์เนต / ออกจากระบบ / รีสตาร์ต / ปิดเครื่อง (สามอย่างสุดท้ายจะขอยืนยันก่อน)
- **หน่วงเวลา** — เพียงรอ N วินาทีก่อนขั้นตอนถัดไป
- **กลุ่มการทำงาน** — รันกลุ่มการทำงานที่กำหนดไว้; ตั้งจำนวนครั้งที่ทำซ้ำเพื่อวนซ้ำทั้งกลุ่ม

> **การหน่วงเวลาเริ่มต้น** (แท็บการตั้งค่า, เฉพาะตอนบูต): รอเป็นจำนวนวินาทีคงที่หลังล็อกอิน เพื่อให้ "พายุการล็อกอิน" (การแย่งใช้ดิสก์/CPU จากทุกโปรแกรมที่เริ่มอัตโนมัติ) ผ่านไปก่อนที่รายการจะรัน; การรันซ้ำด้วยตนเองไม่ได้รับผลกระทบ เพิ่มค่านี้ (0–600 วินาที) หากสิ่งต่าง ๆ เริ่มเร็วเกินไป

> **หยุดได้ทุกเมื่อ** — Tray → **หยุดการทำงานที่กำลังรัน** หรือ **คีย์ลัดฉุกเฉิน** ส่วนกลาง (ตั้งค่าที่แท็บการตั้งค่า; ค่าเริ่มต้น `Ctrl+Alt+Q`) สิ่งที่กำลังรันจะหยุดหลังจากการกระทำปัจจุบัน; การรอที่ยาวนาน (การหน่วงเวลาเริ่มต้น, การรอหน้าต่าง) จะถูกขัดจังหวะทันที

### การเตือน

ตั้ง **เวลา** (หรือสลับเป็น **เมื่อล็อกอิน**), **การเกิดซ้ำ** (วันในสัปดาห์ / ทุก ๆ N วัน / รายเดือน) และ **ข้อความ**; เลือกอ่านออกเสียงได้ การเตือนที่มีการกระทำ **เมื่อกดใช่** (รันโปรแกรม / เปิดไฟล์ / URL / รันกลุ่มการทำงาน) จะเด้งกล่องโต้ตอบ **ใช่ / ไม่ใช่** พร้อมปุ่ม **เลื่อนเตือน** (ค่าเริ่มต้น 10 นาที, เมนู ▾ 5–60 นาที); ส่วนที่เหลือจะเลื่อนขึ้นมาเป็น **การ์ดการเตือน** ที่มุมจอ (ปิดอัตโนมัติหลังจากจำนวนวินาทีที่ตั้งค่าไว้, **0 = ค้างอยู่จนกว่าคุณจะปิดเอง**) คุณยังสามารถตั้ง **กลุ่มการทำงานแบบเงียบ** — รันกลุ่มตรงเวลาโดยไม่มีป๊อปอัป

ขั้นสูง: **ปิดอัตโนมัติ**, **การเตือนซ้ำแบบจี้** (เด้งซ้ำทุก N นาทีจนถึงกำหนดเส้นตาย), **การหน่วงเวลาหลังทริกเกอร์ + การสุ่มเหลื่อมเวลา**, **ช่วงผ่อนผัน** (จับการเตือนที่พลาดไปจากการปิดเครื่อง/สลีปช่วงสั้น ๆ), **ตามเก็บหากพลาด** (เด้งซ้ำครั้งเดียวหลังจากไฮเบอร์เนต/ปิดเครื่องทำให้ข้ามไป) และ **วันอ้างอิง** สำหรับทุก ๆ N วัน (**เลือกวันที่**) "เด้งไปแล้ววันนี้" และ "เลื่อนเตือนจนถึง" จะคงอยู่แม้รีสตาร์ต (`clockwork.state.json`) ดังนั้นการเลื่อนเตือนจึงคงอยู่ข้ามการรีสตาร์ต และไม่มีอะไรเด้งซ้ำซ้อน

ต้องการโฟกัสหรือเข้าประชุมใช่ไหม? Tray มีตัวเลือก **หยุดการเตือนพักไว้ 1 / 2 / 4 ชั่วโมง** (ห้ามรบกวน): ทุกอย่าง (รวมถึงกลุ่มแบบเงียบ) จะถูกระงับและกลับมาทำงานเองอัตโนมัติเมื่อครบเวลา

### รายการเริ่มต้นของระบบ

แสดง **ทุกอย่างที่เริ่มทำงานอัตโนมัติ** (คีย์ Run ในรีจิสทรี, โฟลเดอร์ Startup, งานที่ตั้งเวลาไว้) ยกเลิกการติ๊ก **เปิดใช้งาน** เพื่อปิดรายการ — **ปิดใช้งาน ไม่ได้ลบ; ติ๊กกลับเพื่อกู้คืน** (มีผลทันที) รายการที่ทำเครื่องหมาย **ต้องใช้สิทธิ์ผู้ดูแลระบบ** จะขอเปิดใหม่แบบยกระดับสิทธิ์ รายการของระบบ / นโยบาย / ครั้งเดียว (Group-Policy Run, RunOnce, Winlogon, Active Setup) ไม่สามารถสลับได้ตามปกติ และ **ถูกซ่อนไว้โดยค่าเริ่มต้น** — ติ๊ก **แสดงรายการของระบบ / อ่านอย่างเดียว** เพื่อดู (จะเป็นสีเทา) **รับช่วงเข้ารายการเริ่มต้น** จะส่งมอบรายการให้ Clockwork (เฉพาะคีย์ Run ในรีจิสทรีและรายการในโฟลเดอร์ Startup) **ตัวกรอง** ด้านบนค้นหาตามชื่อ / คำสั่ง; วางเมาส์เหนือคำสั่งที่ถูกตัดให้สั้นเพื่ออ่านแบบเต็ม

### กลุ่มการทำงาน

รวมการทำงานหลายอย่างไว้ในกลุ่มที่นำกลับมาใช้ซ้ำได้ **เพิ่ม ▾** จะเริ่มกลุ่มจาก **เทมเพลตในตัว** (โฟกัส / ประชุม / สรุปงาน / เข้านอน / ลุกจากโต๊ะ / ภาพหน้าจอ) — ปรับชื่อโปรเซสแล้วบันทึก กลุ่มจะ **กำหนดเฉพาะการกระทำ** เท่านั้น; ทริกเกอร์ได้สี่วิธี: จาก Tray (**รัน: <กลุ่ม>**), **คีย์ลัดส่วนกลาง**, เป็น **ขั้นตอนกลุ่มการทำงาน** ในรายการเริ่มต้น (ตอนบูต), หรือจากการเตือน (**เมื่อกดใช่ / กลุ่มแบบเงียบ**) กลุ่มหนึ่งจะรันได้เพียงชุดเดียวในแต่ละครั้ง; ขั้นตอน **ข้อความ** สามารถทำหน้าที่เป็นด่านยืนยันได้ (การตอบ **ไม่ใช่** จะยกเลิกส่วนที่เหลือ)

> **คีย์ลัดส่วนกลาง** — ในตัวแก้ไขกลุ่ม คลิกที่กล่องคีย์ลัดแล้วกดคีย์ลัด (เช่น `Ctrl+Alt+F`) เพื่อรันกลุ่มนั้นจากที่ใดก็ได้ โดยไม่ต้องเปิดเมนู Esc ยกเลิก, Delete ล้างค่า กลุ่มที่ปิดใช้งานจะปล่อยคีย์ผสมของมัน; คีย์ผสมที่ระบบสงวนไว้ (Alt+F4, Ctrl+Shift+Esc…) และคีย์ผสมที่ถูกใช้ไปแล้วโดยกลุ่มอื่นหรือคีย์ลัดฉุกเฉินจะถูกปฏิเสธพร้อมการแจ้งเตือน

### การตั้งค่า

**การหน่วงเวลาเริ่มต้น** (0–600 วินาที, เฉพาะตอนบูต), **เริ่มแบบย่อลง Tray**, **คีย์ลัดฉุกเฉิน** (คลิกกล่องแล้วกดคีย์ลัดของคุณ; Esc ยกเลิก, Delete ล้างค่า; ค่าเริ่มต้น `Ctrl+Alt+Q`) และ **ภาษาของ UI** (จีนตัวย่อ, อังกฤษ, 日本語 และอีก 15 ภาษา — รวม 18 ภาษา; การสลับจะรีสตาร์ตแอปเพื่อให้มีผล)

## เคล็ดลับ

- **ดับเบิลคลิกที่แถวเพื่อแก้ไข** เมื่อกรอกเส้นทาง / โปรเซส / คีย์ลัด / วันที่ คุณไม่ต้องพิมพ์เองด้วยมือ: **เรียกดู…**, **เลือก…** (ตัวเลือกโปรเซสที่ค้นหาได้), **บันทึก** และ **เลือกวันที่**
- การดับเบิลคลิก `Clockwork.exe` เพียงเปิดการตั้งค่าเท่านั้น — มัน **ไม่** รันรายการเริ่มต้นทันที; ให้ใช้ **รันรายการเริ่มต้นอีกครั้ง** จาก Tray แทน
- **เปิดมันแบบปกติ** (ดับเบิลคลิก / Tray / งานที่ตั้งเวลาไว้) ตัวเปิดแบบแซนด์บ็อกซ์ / สิทธิ์ลดทอนบางตัวจะบล็อกการเรียกระดับต่ำ ดังนั้นการส่งคีย์ / การกระทำกับหน้าต่าง / สลับไปหน้าต่างเดิมถ้ากำลังรันอยู่ / การส่งข้อความไปยังโปรเซส / ระดับเสียง อาจไม่ทำงาน (คุณจะได้รับการแจ้งที่ชัดเจน; "เปิดโปรแกรม" ธรรมดาไม่ได้รับผลกระทบ)
- คอนฟิกของคุณคือ `clockwork.settings.json` (อยู่ในเครื่องเท่านั้น) ลบมันเพื่อรีเซ็ตกลับเป็นตัวอย่าง สถานะการเตือนคือ `clockwork.state.json` (อยู่ในเครื่องเช่นกัน; ลบได้อย่างปลอดภัย)
- การเพิ่มขั้นตอน `.ahk` ต้องติดตั้ง AutoHotkey ไว้ คีย์ลัดส่วนกลาง / การขยายข้อความอยู่นอกขอบเขต — นั่นเป็นจุดแข็งของ AutoHotkey

## สำหรับนักพัฒนา

C#/.NET WPF; ซอร์สโค้ดอยู่ใน `app/` (ต้องใช้ .NET 10 SDK) เลเยอร์: `Core/` ตรรกะล้วน · `Native/` Win32 interop · `Engine/` การดำเนินการ · `ViewModels/` + `Views/` UI · `I18n/` + `Resources/` การแปลภาษา (neutral = ซอร์สภาษาจีน, หนึ่ง `Strings.<code>.resx` satellite ต่อภาษา)

- รันเทสต์ (xUnit):
  ```powershell
  dotnet test app.Tests/Clockwork.Tests.csproj
  ```
- บิลด์ไฟล์เดียวแบบ self-contained (single-file / self-contained / compression ถูกตั้งค่าไว้ใน csproj):
  ```powershell
  dotnet publish app/Clockwork.csproj -c Release -r win-x64
  ```
  ผลลัพธ์: `app/bin/Release/net10.0-windows/win-x64/publish/Clockwork.exe`
- **CI / releases** (GitHub Actions): การ push / PR จะบิลด์และรันเทสต์ทั้งหมดบน Windows runner; การ push แท็ก `v*` (เช่น `v2.0.0`) จะบิลด์, ประทับเวอร์ชันไฟล์จากแท็ก, สร้าง GitHub Release และแนบ `Clockwork.exe`

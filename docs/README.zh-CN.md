> 365 开源计划 #020 · Windows 托盘小工具：开机启动 · 定时提醒 · 系统启动项 · 动作组

![Clockwork](../assets/social-card.png)

一个给 Windows 用的托盘小工具，帮你**管好开机与日常这几件事**：

- 🚀 **我的启动清单** —— 开机后自动、按顺序打开常用软件（可设管理员权限、先后延时、仅某些星期 / 仅 N 点前、窗口风格、已开则激活、备用路径），还能顺手做几件杂事（关掉或激活某些窗口、发送按键 / 发送文本、调音量等）。
- ⏰ **定时提醒** —— 到点弹提醒，可朗读、按「星期 / 每 N 天 / 每月」周期、也能「登录时」触发；点「是」还能运行程序 / 打开文件（如音乐）/ 开网页 / 跑一个动作组。
- 🧹 **系统启动项** —— 把电脑里**所有开机自启的软件**列出来，一键关掉不需要的（只禁用、不删除，随时恢复），还能一键「纳入」到自己的启动清单来接管。
- 🎛️ **动作组** —— 把一串动作打包成可复用的「组」（专注 / 会议 / 收工 / 睡前……），在托盘、**全局热键**、启动清单或提醒里一键触发；内置常用模板。

免安装、绿色单文件夹、所有设置点点鼠标就能改；暗色界面、高分屏自适应。界面提供 **18 种语言**，首次运行默认跟随系统显示语言。

> 📖 **详细使用说明：** [English](USAGE.md) · [中文](USAGE.zh-CN.md)

## 运行要求

- Windows 10 / 11（x64）
- 无需额外安装：自包含单文件 `Clockwork.exe`，内置 .NET 运行时，开箱即用。

## 怎么开始用

1. 到 [Releases](https://github.com/rockbenben/Clockwork/releases) 下载最新的 `Clockwork.exe`，放进任意文件夹（绿色免安装，放哪都行）。想自己构建见下方 **给开发者**。
2. 双击 **`Clockwork.exe`** 打开设置窗口。
   - **首次打开**会自动载入一份**示例配置**（演示各种玩法的启动 / 提醒 / 动作组），照着改成你自己的即可。你的设置存在同文件夹的 `clockwork.settings.json`，只在本机、不入库。
3. 想每次开机自动运行：到 **「设置」** 页点「开机自启」按钮开启（用管理员权限注册计划任务，避免开机弹一堆授权框）。

> 平时它安静待在右下角托盘。双击托盘图标打开窗口；点窗口关闭按钮只是收起到托盘，不会退出。彻底退出用托盘右键「退出」。

## 五个标签页

### 我的启动清单

一条**有序步骤列表**，开机时从上到下依次执行。点「新增 ▾」选类型，可任意增删、上下移；每步可单独启用 / 禁用、设「执行后延时」、设「重复次数」（循环 N 次），并加条件「仅某些星期 / 仅 N 点前」。步骤类型：

- **启动程序** —— 目标（可「浏览…」选文件）/ 参数 / 工作目录（留空 = 目标所在目录）/ 管理员。目标可填 `.exe`、文档、快捷方式、网址；`.ps1` 会自动用 PowerShell 运行。进阶：**窗口风格**（最小化 / 最大化 / 隐藏）、**已在运行则激活**（把它带到最前、不重复启动；进程名可「选择…」）、**备用路径**（每行一条完整路径，取第一个存在的——多设备安装路径不一致时用）。
- **发送按键** —— 如 Win+D、Alt+K、Ctrl+Enter、F5（可点「捕获」直接按下快捷键录入）。
- **发送文本** —— 往当前焦点窗口逐字输入一段文本（也可指定「目标进程」，可「选择…」）。
- **音量** —— 静音 / 取消静音 / 设为音量。
- **窗口动作** —— 按进程名（可「选择…」，带搜索）关闭 / 最小化 / 最大化 / 带到最前面 / 置前并发送按键；慢启动应用可设「等窗口出现最多 N 秒」。
- **系统命令** —— 显示桌面 / 锁屏 / 息屏 / 清空回收站 / 清空剪贴板 / 打开设置 / 任务管理器 / 截图 / 睡眠 / 休眠 / 注销 / 重启 / 关机（后三项执行前弹确认框）。
- **延时** —— 单纯等待 N 秒再继续。
- **动作组** —— 引用一个已定义好的动作组；设「重复次数」即可整组循环。

> **开机延迟**（设置页，仅开机自启）：登录后先固定等这么多秒，让「登录风暴」（磁盘 / CPU 被一堆自启抢占）过峰，再跑清单；手动「重新运行」不受影响。开机太早、有软件没起来就把它调大（0–600 秒）。

> **随时叫停** —— 托盘「停止正在运行的动作」，或按全局**急停键**（设置页设，默认 `Ctrl+Alt+Q`）。正在跑的会在当前动作后立刻停下；开机延迟、等窗口出现等长等待也会被立即打断。

### 定时提醒

设**时间**（或改「登录时」）、**周期**（星期 / 每 N 天 / 每月）、**提醒文字**，可选朗读。配了「点是后」动作（运行程序 / 打开文件 / 开网页 / 跑动作组）的会弹「是 / 否」窗、还带「稍后」（默认 10 分钟，▾ 可选 5–60 分钟）；其余走右下角**提醒卡片**（不抢视线，显示时长由「自动关闭」秒数决定，**0 = 常驻到你点掉**）。也可设「**静默动作组**」——到点不弹窗、直接跑指定动作组。

进阶：**自动关闭**、**重复催促**（每 N 分钟再弹、直到某时刻）、**到点后延迟 + 随机延迟**、**宽限**（到点恰好关机 / 睡眠时补弹）、**错过必补**（休眠 / 关机整段错过时下次补弹一次）、「每 N 天」的**起算日**（可「选日期」）。「当天已弹 / 稍后到几点」会落盘（`clockwork.state.json`），跨重启不丢、也不重弹。

要专注或开会时，托盘可「**暂停提醒** 1 / 2 / 4 小时」（勿扰）：期间所有提醒（含静默组）都不触发，到时自动恢复。

### 系统启动项

列出电脑里**所有开机自启**的项（注册表 Run 键、启动文件夹、计划任务）。取消「启用」即禁用——**只禁用、不删除，随时勾回来恢复**（即时生效）。带「需管理员」的项会提示以管理员身份重开。系统 / 策略 / 一次性项（组策略 Run、RunOnce、Winlogon、Active Setup）无法常规开关、**默认不显示**——勾「显示系统 / 只读项」才列出（置灰）。「纳入启动清单」把某项交给本工具接管（仅支持注册表 Run 键与启动文件夹项）。顶部**过滤框**按名称 / 命令快速筛；命令太长悬停可看全文。

### 动作组

把一串动作打包成可复用的「组」。「新增 ▾」可从**内置模板**（专注 / 会议 / 收工 / 睡前 / 离开一下 / 截图标注）创建，改进程名再保存。组本身**只定义动作**，四种触发：托盘「运行：某组」、**全局热键**、启动清单里加「动作组」步骤、提醒挂「点是后 / 静默动作组」。同组同时只跑一份；「消息」步骤可当**确认闸门**（答「否」中止整组）。

> **全局热键** —— 在组编辑器里点热键框、按下快捷键（如 `Ctrl+Alt+F`），即可在任何地方运行该组，无需打开菜单。Esc 取消、Delete 清空。停用的组会释放其组合键；系统保留的组合键（Alt+F4、Ctrl+Shift+Esc……）以及已被其它组或急停键占用的组合键会被拒绝并提示。

### 设置

**开机延迟**（0–600 秒，仅开机自启）、**启动时最小化到托盘**、**急停键**（点输入框按下即录入，Esc 取消、Delete 清空；默认 `Ctrl+Alt+Q`）、**界面语言**（简体中文、English、日本語 等共 18 种，切换后自动重启生效）。

## 小提示

- 列表里**双击条目即编辑**。填路径 / 进程 / 组合键 / 日期不用硬记：**「浏览…」「选择…」（带搜索）「捕获」「选日期」**。
- 双击 `Clockwork.exe` 只开设置窗口，**不会**立刻跑启动清单；想立刻跑用托盘「重新运行启动清单」。
- **请用正常方式启动**（双击 / 托盘 / 计划任务）。若用某些沙箱 / 受限权限的启动器（如 Lucy）打开，Windows 会限制部分底层调用，发送按键 / 窗口动作 / 已运行则激活 / 发送文本到指定进程 / 音量 等可能不可用（会给清楚提示，不影响单纯「启动程序」）。
- 个人设置在 `clockwork.settings.json`（仅本机）。删掉它重开即可恢复示例。提醒状态在 `clockwork.state.json`（同样只在本机，删掉无妨）。
- 加 `.ahk` 步骤需装 AutoHotkey。全局热键 / 文本展开不在本工具范围（那是 AutoHotkey 的强项）。

## 给开发者

C#/.NET WPF，源码在 `app/`（需 .NET 10 SDK）。分层：`Core/` 纯逻辑 · `Native/` Win32 互操作 · `Engine/` 执行 · `ViewModels/` + `Views/` 界面 · `I18n/` + `Resources/` 多语言（中性 = 中文源，`Strings.<code>.resx` 每语言一份卫星程序集）。

- 跑测试（xUnit）：
  ```powershell
  dotnet test app.Tests/Clockwork.Tests.csproj
  ```
- 打包自包含单文件 exe（单文件 / 自包含 / 压缩等属性已写在 csproj 里）：
  ```powershell
  dotnet publish app/Clockwork.csproj -c Release -r win-x64
  ```
  产物在 `app/bin/Release/net10.0-windows/win-x64/publish/Clockwork.exe`。
- **CI / 发版**（GitHub Actions）：push / PR 自动在 Windows runner 上构建并跑全部测试；推送 `v*` 标签（如 `v2.0.0`）会自动构建、以标签号作文件版本、创建 GitHub Release 并附上 `Clockwork.exe`。

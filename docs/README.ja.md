> 365 オープンソース計画 #020 · Windows のトレイツール：スタートアップランチャー · リマインダー · システムのスタートアップ項目 · アクショングループ

![Clockwork](../assets/social-card.png)

一日のはじめに PC でおこなう決まりきった作業を引き受けてくれる、小さな Windows トレイツールです：

- 🚀 **スタートアップリスト** — ログイン時に毎日使うアプリを順番に自動で開き（ステップごとに管理者権限、遅延、特定の曜日のみ / N 時より前のみ、ウィンドウスタイル、実行中なら前面化、フォールバックパスを設定可能）、ついでにちょっとした雑務もこなします（ウィンドウを閉じる・フォーカスする、キー入力 / テキストを送る、音量を設定する…）。
- ⏰ **リマインダー** — 時間どおりにリマインダーをポップアップ；読み上げ；曜日 / N 日ごと / 毎月で繰り返し；または「ログイン時」に発火。**はい** をクリックすると、プログラムの実行、ファイル（音楽など）や URL を開く、アクショングループの実行ができます。
- 🧹 **システムのスタートアップ項目** — **PC で自動起動するすべての項目**を一覧表示し、不要なものをオフに（削除ではなく無効化——いつでも戻せます）。ワンクリックで項目を「引き取り」、自分のスタートアップリストに取り込めます。
- 🎛️ **アクショングループ** — 一連のアクションを再利用可能なグループにまとめ（集中 / 会議 / 締め / 就寝…）、トレイ・**グローバルホットキー**・スタートアップリスト・リマインダーからワンクリックで発火。組み込みテンプレートも用意。

インストール不要、完全ポータブルな単一フォルダ、すべてマウスで設定可能；ダーク UI、高 DPI 対応。

> 📖 **詳しい使い方：** [English](USAGE.md) · [中文](USAGE.zh-CN.md)

## 動作要件

- Windows 10 / 11（x64）
- インストール不要：.NET ランタイムを同梱した自己完結型の単一ファイル `Clockwork.exe`。

## はじめに

1. [Releases](https://github.com/rockbenben/Clockwork/releases) から最新の `Clockwork-<バージョン>.zip` をダウンロードして展開すると、中に単一の `Clockwork.exe` が入っています；それを任意のフォルダに置きます（ポータブル——どこに置いても OK）。自分でビルドする場合は下の **開発者向け** を参照。
2. **`Clockwork.exe`** をダブルクリックして設定ウィンドウを開きます。
   - **初回起動**では、スタートアップ一覧とリマインダーにいくつかの **サンプル** が読み込まれるので、自分用に手直しできます——いずれも最初はチェックが外れているので、自分でチェックを入れるまで何も実行されません。設定は exe と同じ場所の `clockwork.settings.json` に保存されます——ローカル限定で、コミットされることはありません。
3. 毎回の起動時に実行するには：**設定** タブで **ログイン時に起動** をクリック（管理者権限でスケジュールされたタスクを登録するので、起動時に UAC の確認が大量に出ることはありません）。

> 静かにトレイに常駐します。トレイアイコンをダブルクリックするとウィンドウが開きます；ウィンドウの閉じるボタンはトレイに隠すだけです。完全に終了するには、トレイの右クリック **終了** から。

## 5 つのタブ

### スタートアップリスト

ログイン時に上から下へ実行される**順序付きのステップリスト**です。**追加 ▾** をクリックして種類を選択；自由に追加 / 削除 / 並べ替えができ、各ステップは有効 / 無効の切り替え、**ステップ後の遅延**、**繰り返し回数**（N 回ループ）、条件（**特定の曜日のみ / N 時より前のみ**）を設定できます。ステップの種類：

- **プログラムを起動** — ターゲット（**参照…** でファイルを選択）/ 引数 / 作業ディレクトリ（空欄 = ターゲットのフォルダ）/ 管理者。ターゲットは `.exe`、ドキュメント、ショートカット、URL のいずれでも可；`.ps1` は PowerShell 経由で実行されます。詳細設定：**ウィンドウスタイル**（最小化 / 最大化 / 非表示）、**すでに実行中なら前面化**（再起動せず前面に持ってくる；プロセス名は **選択…** で）、**フォールバックパス**（1 行に 1 つのフルパス；最初に存在するものが使われます——マシンごとにインストール先が異なるときに便利）。
- **キーを送信** — 例：Win+D、Alt+K、Ctrl+Enter、F5（**キャプチャ** でショートカットを実際に押して記録）。
- **テキストを送信** — フォーカスされたウィンドウ（または **選択…** で選んだ **ターゲットプロセス**）に文字列を入力。
- **音量** — ミュート / ミュート解除 / レベル設定。
- **ウィンドウ操作** — プロセス名で（**選択…**、検索可能）：閉じる / 最小化 / 最大化 / 前面化 / 前面化してキー送信；遅いアプリは**ウィンドウが現れるまで最大 N 秒待機**できます。
- **システムコマンド** — デスクトップの表示 / ロック / モニターをオフ / ごみ箱を空にする / クリップボードをクリア / 設定を開く / タスクマネージャー / スクリーンショット / スリープ / 休止状態 / サインアウト / 再起動 / シャットダウン（最後の 3 つは先に確認します）。
- **遅延** — 次のステップの前に N 秒待つだけ。
- **アクショングループ** — 定義済みのアクショングループを実行；繰り返し回数を設定するとグループ全体をループできます。

> **起動遅延**（設定タブ、起動時のみ）：ログイン後に一定の秒数待ち、「ログインストーム」（すべての自動起動によるディスク / CPU の競合）が収まってからリストを実行します；手動での再実行には影響しません。開始が早すぎる場合は上げてください（0〜600 秒）。

> **いつでも停止** — トレイ → **実行中のアクションを停止**、またはグローバルな **緊急停止ホットキー**（設定タブで設定；既定は `Ctrl+Alt+Q`）。実行中のものは現在のアクションの後で停止します；長い待機（起動遅延、ウィンドウ待ち）はただちに中断されます。

### リマインダー

**時刻**（または **ログイン時** に切り替え）、**繰り返し**（曜日 / N 日ごと / 毎月）、**テキスト**を設定；必要なら読み上げも。**「はい」で実行するアクション**（プログラム実行 / ファイルを開く / URL / アクショングループ実行）を持つリマインダーは、**スヌーズ** ボタン付きの **はい / いいえ** ダイアログをポップアップします（既定 10 分、▾ メニューで 5〜60 分）；それ以外は隅に**リマインダーカード**としてスライドイン（設定した秒数後に自動で閉じる、**0 = 手動で閉じるまで表示**）。**サイレントアクショングループ**も設定できます——ポップアップなしで時間どおりにグループを実行。

詳細設定：**自動クローズ**、**繰り返し催促**（期限まで N 分ごとに再ポップアップ）、**発火後の遅延 + ランダムなゆらぎ**、**猶予**（短時間のシャットダウン / スリープで逃した発火を拾う）、**逃したら追いつく**（休止状態 / シャットダウンで飛んだ後に一度だけ再発火）、そして N 日ごとの**基準日**（**日付を選択**）。「本日発火済み」と「スヌーズ中（〜まで）」は再起動をまたいで保持され（`clockwork.state.json`）、スヌーズは再起動後も引き継がれ、二重発火もしません。

集中したい、会議に入りたい？トレイには **リマインダーを 1 / 2 / 4 時間停止**（応答不可）があります：すべて（サイレントグループを含む）が抑制され、時間になると自動で再開します。

### システムのスタートアップ項目

**自動起動するすべての項目**を一覧表示します（レジストリの Run キー、スタートアップフォルダ、スケジュールされたタスク）。**有効** のチェックを外すと項目をオフにできます——**削除ではなく無効化；再チェックで復元**（ただちに反映）。**管理者が必要** と表示された項目は昇格して再起動するよう促します。システム / ポリシー / 一回限りの項目（グループポリシー Run、RunOnce、Winlogon、Active Setup）は手を加えられず、**既定では非表示**です——**システム / 読み取り専用の項目を表示** にチェックすると表示されます（グレー表示）。行を**右クリック**すると **起動リストに取り込む**（項目を Clockwork に渡します；レジストリの Run キーとスタートアップフォルダの項目のみ）または **システムから削除**（エントリを完全に削除します——先に確認があり、元に戻せません；チェックを外すほうが取り消し可能な選択肢です）が使えます。上部の**フィルター**で名前 / コマンドを検索できます；切り詰められたコマンドはマウスを乗せると全文を読めます。

### アクショングループ

アクションを再利用可能なグループにまとめます。**追加 ▾** は**組み込みテンプレート**（集中 / 会議 / 締め / 就寝 / 離席 / スクリーンショット）から始めます——プロセス名を調整して保存するだけ。グループは**アクションを定義するだけ**；発火方法は 4 通り：トレイから（**実行：<グループ>**）、**グローバルホットキー**、スタートアップリストの**アクショングループステップ**として（起動時）、またはリマインダーから（**「はい」で / サイレントグループ**）。1 つのグループは同時に 1 つしか実行されません；**メッセージ** ステップは確認ゲートとして機能します（**いいえ** と答えると残りを中止）。

> **グローバルホットキー** — グループエディタでホットキーのボックスをクリックし、ショートカット（例：`Ctrl+Alt+F`）を押すと、メニューを開かずにどこからでもそのグループを実行できます。Esc でキャンセル、Delete でクリア。無効なグループは組み合わせを解放します；システム予約の組み合わせ（Alt+F4、Ctrl+Shift+Esc…）や、他のグループまたは緊急停止ホットキーがすでに使用中の組み合わせは、通知とともに拒否されます。

### 設定

**起動遅延**（0〜600 秒、起動時のみ）、**起動時にトレイに最小化**、**緊急停止ホットキー**（ボックスをクリックしてショートカットを押す；Esc でキャンセル、Delete でクリア；既定は `Ctrl+Alt+Q`）、そして **UI 言語**（簡体字中国語、English、日本語 ほか——全 18 種；切り替えると適用のためアプリが再起動します）。

**設定エクスポート / 設定インポート** — 設定一式を別の PC に移したり、バックアップとして保管したりできます。エクスポートは `clockwork.settings.json` のコピーを好きな場所に書き出します；インポートは**すべて**（スタートアップリスト / リマインダー / アクショングループ / 設定）を置き換えるため、先に確認したうえで現在の設定を `clockwork.settings.json.bak` にバックアップし、適用のためアプリを再起動します。

## ヒント

- **行をダブルクリックして編集**。パス / プロセス / ショートカット / 日付を入力するとき、手打ちする必要はありません：**参照…**、**選択…**（検索可能なプロセスピッカー）、**キャプチャ**、**日付を選択** があります。
- **複製**（リマインダー / アクショングループのタブ）は選択中の行をそのすぐ下に複製します——よく似た設定をもう一度作り直すより速く、複製したグループの名前は「…（コピー）」になります。
- **削除は必ず先に確認します**——リストの行、グループエディタ内のステップ、システムのスタートアップ項目、どこでも同じです。
- `Clockwork.exe` のダブルクリックは設定を開くだけで、スタートアップリストをすぐに実行**しません**；実行するにはトレイの **スタートアップリストを再実行** を使ってください。
- **通常どおり起動してください**（ダブルクリック / トレイ / スケジュールされたタスク）。一部のサンドボックス / 権限を落としたランチャーは低レベル呼び出しをブロックするため、キー送信 / ウィンドウ操作 / 実行中なら前面化 / プロセスへのテキスト送信 / 音量が動作しないことがあります（明確な通知が出ます；単純な「プログラムを起動」は影響を受けません）。
- 設定は `clockwork.settings.json`（ローカル限定）。削除するとサンプルにリセットされます。リマインダーの状態は `clockwork.state.json`（これもローカル；削除しても安全）。
- `.ahk` ステップの追加には AutoHotkey のインストールが必要です。グローバルホットキー / テキスト展開は対象外です——それは AutoHotkey の得意分野です。

## 開発者向け

C#/.NET WPF；ソースは `app/`（.NET 10 SDK が必要）。レイヤー：`Core/` 純粋ロジック · `Native/` Win32 インターオペ · `Engine/` 実行 · `ViewModels/` + `Views/` UI · `I18n/` + `Resources/` ローカライズ（ニュートラル = 中国語ソース、言語ごとに 1 つの `Strings.<code>.resx` サテライト）。

- テストを実行（xUnit）：
  ```powershell
  dotnet test app.Tests/Clockwork.Tests.csproj
  ```
- 自己完結型の単一ファイル exe をビルド（単一ファイル / 自己完結 / 圧縮は csproj で設定済み）：
  ```powershell
  dotnet publish app/Clockwork.csproj -c Release -r win-x64
  ```
  出力：`app/bin/Release/net10.0-windows/win-x64/publish/Clockwork.exe`。
- **CI / リリース**（GitHub Actions）：push / PR で Windows ランナー上でビルドし、すべてのテストを実行；`v*` タグ（例：`v2.0.0`）を push すると、ビルドし、タグからファイルバージョンを刻印し、GitHub Release を作成して `Clockwork-<タグ>.zip`（`Clockwork.exe` を同梱）を添付します。
